Help operativo

Obiettivo del plugin
Trasformare gli elementi del progetto QGIS in un computo metrico estimativo: ogni feature richiama un codice voce di prezziario e la sua quantità viene calcolata dalla geometria o letta da un campo manuale.

1. Flusso consigliato

  1. Apri la scheda Info e crea il progetto demo se vuoi partire da un esempio già pronto.
  2. Vai in Prezziari e usa prima il Catalogo regionale: per ogni regione vedi subito se il dataset è pronto, solo scaricabile oppure da compilare manualmente.
  3. Se disponibile, importa il dataset locale pronto. In alternativa usa CSV/XLSX, un DCF leggibile, link salvati o download ufficiale.
  4. Se il formato ufficiale non è direttamente importabile, crea il template manuale regionale e compila il CSV con la struttura richiesta.
  5. Vai in Misurazioni, scegli il layer e verifica il mapping dei campi.
  6. Clicca Calcola computo dal layer.
  7. Vai in Computo per completare intestazione, logo, riferimenti e mappa del report.
  8. Vai in Contabilità per generare SAL, certificato di pagamento, registro, libretto e giornale lavori.
Catalogo regionale
Il catalogo mostra tre stati:
  • Dataset pronto: il plugin include già un file normalizzato e importabile.
  • Conversione manuale: il portale ufficiale espone un file consultabile, ma non ancora affidabile per import diretto.
  • Portale censito: la regione è presente, ma il dataset non è ancora bundle locale.
Il file scaricato viene sempre salvato in una cartella locale accessibile dal plugin.
Supporto DCF
Il plugin prova a leggere un file .dcf solo se è realmente interpretabile come:
  • contenitore ZIP con CSV/XLSX interni;
  • database SQLite con tabelle riconoscibili;
  • tabella testuale uniforme.
Se il DCF è proprietario o non documentato, il plugin lo rifiuta e chiede la conversione in CSV/XLSX.

2. Formato atteso dei layer

Campo Obbligatorio Uso
item_code Sì Codice della voce presente nel prezziario attivo.
descrizione No Descrizione specifica della feature; se vuota viene usata quella del prezziario.
categoria No Raggruppamento di capitoli o lavorazioni.
quantita_man No Se compilata, sostituisce il calcolo geometrico automatico.
coefficiente No Moltiplicatore numerico applicato alla quantità finale.
unita_override No Forza l'unità di misura della singola feature.
larghezza_m No Permette di derivare superfici da linee e volumi da elementi lineari.
altezza_m No Permette di derivare volumi da linee o punti.
spessore_m No Permette di derivare volumi da poligoni o linee.
pezzi No Permette quantità a corpo più precise rispetto al semplice conteggio feature.
note No Annotazioni riportate nel dettaglio del report.

4. Regole di misura automatiche

  • mq e ha: misura dell'area.
  • mq su layer lineari: se presente larghezza_m, usa lunghezza x larghezza.
  • ml e km: misura della lunghezza; per i poligoni usa il perimetro.
  • cad: una unità per feature, oppure il valore del campo pezzi.
  • mc: area x spessore oppure lunghezza x larghezza x altezza/spessore.

5. Report completo

Nella scheda Computo puoi impostare:
  • logo del documento;
  • titolo, sottotitolo e organizzazione;
  • riferimenti multipli con pulsante Add +;
  • piè di pagina / note finali;
  • inclusione della mappa corrente nel PDF.
Il profilo viene salvato nel database locale del plugin.

6. Avvisi importanti

  • Usa un CRS metrico proiettato per evitare risultati incoerenti.
  • Se il plugin segnala dati non uniformi, correggi intestazioni o valori numerici prima di importare.
  • Se un codice voce non esiste nel prezziario attivo, il plugin lo segnala e assegna prezzo zero.
